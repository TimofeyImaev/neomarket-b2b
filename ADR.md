# ADR для описания PR (US-B2B-01)

Рассматривали три способа хранить характеристики товара: JSON-поле в таблице
products, отдельная таблица product_characteristics и EAV-схема со справочником
атрибутов. Выбрали отдельную таблицу. Критерии: сложность запросов при
фильтрации по характеристикам и удобство добавления новых характеристик.
JSON-поле завязывает фильтрацию на jsonb-операторы и плохо индексируется по
произвольным ключам, EAV дает гибкость, но тащит справочник и лишние джойны в
каждый запрос. Отдельная таблица дает простые индексируемые запросы
(WHERE name = ? AND value = ?), новые характеристики добавляются вставкой строки
без миграций, а переход к EAV при необходимости останется эволюционным.
